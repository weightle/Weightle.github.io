Mac端hexo部署

安装nvm

使用gitee镜像安装nvm到本地

本地有git直接安装nvm无git优先安装git第二遍安装nvm

‘git clone https://gitee.com/mirrors/nvm.git ~/.nvm && cd ~/.nvm && git checkout `git describe --abbrev=0 --tags`‘

配置nvm环境变量

下载完成 当终端关闭之后在打开nvm失效 需进入 ‘.bash_profile’ 文件设置环境变量

‘vi ~/.bash_profile #进入（i编辑 esc退出 :wq保存）
copy下面的两行并且保存
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh" # This loads nvm
wq保存后执行生效命令
source ~/.bash_profile’

使用zsh环境需单独设置.zshrc文件

‘vi ~/.zshrc (i编辑 esc退出 ：wq保存)
放置
source ~/.bash_profile
wq保存执行生效命令
source ~/.zshrc‘

nvm安装node

‘nvm install V8.8.0
nvm install node’

切换node版本 nvm use x.x.x

hexo安装

‘npm install -g hexo-cli'

报错检查node 卸载重装

本地初始化

’hexo init blog#初始化
cd blog
npm istall‘

注意事项

配置SSH Keys

‘cd ～/.ssh #检查本机ssh密钥
ssh-keygen -t rsa -C "你的邮箱地址" #生成密钥

生成id_rsa.pub隐藏文件（快捷键shift+cmd+.）copy
登陆GitHub Setting->SSH->New SSH Keys 保存退出

卸载已有node和node模块（清环境 第一次安装直接跳过）

‘npm ls -g --depth=0                             # 查看已经安装在全局的模块，以便删除这些全局模块后再按照不同的 node 版本重新进行全局安装
sudo rm -rf /usr/local/lib/node_modules         # 删除全局 node_modules 目录
sudo rm /usr/local/bin/node                     # 删除 node
cd  /usr/local/bin && ls -l | grep "../lib/node_modules/" | awk '{print $9}'| xargs rm       #删除全局 node 模块注册的软链
node -v #检查本地node

cnpm 国内代理

’npm install -g cnpm --registry=http://registry.npm.taotao.org'

同步node模块下安装环境

先卸载后安装的node版本 在重新使用
‘nvm install –reinstall-packages-from=8.8.0#此处版本号根据自己实际安装node版本调整’

hexo部署

hexo -c#清缓存
hexo -s#开启本地服务
hexo -g#保存
hexo -d#上传GitHub
